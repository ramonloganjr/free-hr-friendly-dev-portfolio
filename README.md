# Free HR Friendly Dev Portfolio

A clean and modern portfolio template for developers who want to be easy to understand for HR teams, recruiters, and hiring managers. This website uses a polished, glassy visual style with clear sections that help visitors quickly see experience, projects, and value.

## Latest updates

This version includes a refreshed presentation with:

- a clearer home page flow from introduction to featured work
- a simplified project showcase with six featured projects
- easy to browse sections for projects, about, writing, and contact
- a fast, static build that is simple to customize

## Why this is ideal for HR and recruiters

This portfolio is easier for non-technical hiring managers to review than a complex and complicated portfolio because it presents information in a simple and familiar format. Recruiters can quickly understand what kind of work a developer has done, what tools were used, and which projects are most relevant to a role.

A non-technical hiring manager does not need to decode code, architecture diagrams, or deep engineering detail to make a strong first assessment. This template helps them focus on outcomes, relevance, and communication, which makes it easier to decide whether a candidate is worth a deeper conversation.

## Features

- fast static site experience with Next.js and React
- responsive design with a polished Liquid Glass inspired look
- clear navigation for home, projects, about, writing, and contact
- structured content for featured work, skills, and professional story
- simple customization through the data files in src/lib/data
- built for clarity, trust, and recruiter friendly presentation

## Getting started

```bash
npm install
npm run dev
npm run build
npm start
```

## Project structure

```text
src/
  app/                  # routes for home, projects, solutions, about, writing, and contact
  components/           # reusable UI and page sections
  lib/data/             # typed content for projects, testimonials, clients, and more
public/                 # static assets and placeholder media
```
